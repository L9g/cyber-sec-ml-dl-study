• project4-reading-list.md审阅结论。

  我建议主要从 5 个方向补强。

  1. 增加“合规 / 业务流程”阅读线

  现在列表偏算法和数据集，但 AML 岗位面试经常不是只问 GNN，而是问：模型怎么进入 transaction monitoring workflow，怎
  么减少 false positives，怎么支持 investigator，怎么审计。

  建议新增一节：

  ## C. AML 业务流程、交易监控与可解释性

  可补：

  - Anti-Money Laundering Alert Optimization Using Machine Learning with Graphs
    这篇非常贴合真实银行 AML：规则系统产生 alerts，ML 做 alert triage，用图特征降低 false positive。它报告了在真实银
    行数据上减少大量误报、保持较高 true positive detection 的目标。比纯 Elliptic 更接近金融机构工作流。

  - Machine Learning in Transaction Monitoring: The Prospect of xAI
    这篇适合支撑“为什么 AML 模型必须可解释”。它关注 transaction monitoring 中不同责任方对 xAI 的需求，能帮助你把
    GNNExplainer 从技术展示提升到合规/审计叙事。

  这两篇对项目很有价值，因为它们把项目从“crypto graph classification”拉回“financial crime operations”。

  2. 补充 dynamic / temporal graph 基础论文

  项目四主文档里提到 EvolveGCN，但 reading list 里没有独立列出来。建议补到必读或至少强选读：

  - EvolveGCN: Evolving Graph Convolutional Networks for Dynamic Graphs
    Elliptic 原始论文用过 EvolveGCN，项目四如果要做 temporal split 或动态图，这是直接基础。

  - Temporal Graph Networks for Deep Learning on Dynamic Graphs
    TGN 是动态图学习更通用的框架，适合 Research-grade 版本。你不一定实现，但应该读它来理解 event-based dynamic graph
    和 snapshot-based graph 的差异。

  建议在文档里明确区分：

  Snapshot temporal graph: 每个时间步一张图，例如 Elliptic 的 49 steps。
  Event-based temporal graph: 每笔交易是带 timestamp 的事件，例如 TGN 更适合。

  这个区分很重要，否则“时序图”会写得太泛。

  3. 增加不平衡与半监督图学习

  现在文档最后提到 precision@k / recall@k，但缺少专门的 imbalanced graph learning 支撑。AML 的少数类问题不是普通
  class imbalance，因为图结构会让 oversampling 更复杂。

  建议补：

  - GraphSMOTE: Imbalanced Node Classification on Graphs with Graph Neural Networks
    用来说明不能简单把普通 SMOTE 套到图节点上，因为新合成节点还需要合理的边关系。

  - focal loss / class-balanced loss 相关材料可以作为工程方法，不一定列论文。

  然后在项目实施中明确：
  class_weight / focal loss / threshold tuning / precision@k 是 MVP；GraphSMOTE 是扩展，不要一开始就做。

  4. 可解释性部分需要独立成节

  现在只在最后提到 GNNExplainer。建议新增：

  ## D. Explainable GNN for AML

  至少补：

  - GNNExplainer: Generating Explanations for Graph Neural Networks
    这是图解释的基础论文。项目中可以输出：可疑地址的 ego-subgraph、关键邻居、关键边、关键特征。

  还建议把解释目标写得更业务化：

  不是解释“模型为什么分数高”，而是解释“这个 alert 为什么值得 investigator 优先看”。

  AML 里解释的使用者不是 ML 研究员，而是 compliance analyst / investigator。
