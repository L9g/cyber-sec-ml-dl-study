## 1. ✅「崩塌」是相对断言——每个模型必须自带分布内对照

**判断**：要证「所有模型跨数据集一起崩」(Layeghy A2)，每个模型必须**同时**有 (a) 分布内、(b) 跨数据集两个数字；缺 (a) 不能下"崩塌"结论。

**为什么**：单看跨集 PR-AUC=0.4 兼容两个互斥解释——"它本来就弱"(intrinsic) vs "它强但不迁移"(collapse)。区分唯一办法 = 让每个模型做**自己的对照组**：分布内=control，跨集=treatment，崩塌=within-subject 落差。少了 (a) 就退化成"谁强"的跨模型横比（项目要避开的擂台），且会被一句"你的基线本来就菜"打回。

**复盘注意点**：这个约束**会付钱**——正因坚持 (a)，才暴露 CSE-CIC 分布内连 LightGBM 都只有 0.58，逼我把头条从"所有模型在所有数据集崩"收窄成"**UNSW 锚点：连单特征 stump 都 0.998**"。它是防 over-claim 的闸门。任何"X 崩了/退化了/更差了"的断言，先问：**相对什么基线？同一被试的对照在哪？**

## 2. ⚠️ inf/NaN 容忍度差异 = 模型类内禀属性（但非新发现）

**判断**：LightGBM 容忍 inf/NaN，LogReg/MLP 不容忍（NF-v3 rate 特征 duration=0 除零得 inf）；多模型对比时非树模型需统一加 inf→NaN + 中位数插补 + 标准化。

**为什么**：树只用**序**——inf 在序里有定义（进最大 bin），NaN 走"学到的缺失默认方向"，故对 inf/NaN/异常值/单调变换全不变。线性/NN 算加权和 w·x，`inf×w=inf` → 梯度全 NaN → 优化崩；必须给有限值 + 量级可比。

**复盘注意点（含纠正）**：① 这是 ML **常识**不是新发现——准确说法是"一个常被隐藏的不对称，我把它显式化了"。② 对**讲 benchmark 诚实**的项目仍切题：论文写"对比 A/B/C"时，为让非树模型能跑做的插补/缩放**通常不报告**，于是"同一份数据"其实**不同**（一个被插补缩放过）。③ 树"容忍"= **不报错**，≠ 不受影响（inf 丢进最大 bin 也是建模选择）；中位数插补本身是**会影响结果**的选择，不是中性清洗。

## 3. ✅ `df.mode` 撞名 = 静默的 pandas footgun

**判断**：列名 `"mode"` 用 `df.mode` 取会撞上 DataFrame 的 `.mode()` **方法**，过滤恒空、聚合全 NaN；必须用 `df["mode"]`。

**为什么**：`df.mode` 解析成 bound method 对象，`method == "indist"` 恒 False。`df["mode"]` 走 `__getitem__` 强制按列取，绕开属性查找。

**复盘注意点**：① **不报错只给 NaN**——最阴；靠冒烟里**肉眼看汇总输出**才抓到 → 冒烟别只看 exit code，要看内容。② 程序化代码一律 `df["col"]` 不用 `df.col`（撞名的方法/属性一大把：mode/count/mean/sum/index/columns/size/shape/T…）。③ 根因是列名遮蔽方法——别给列起这种名字（用 `regime`/`split_mode`）。

## 4. ⚠️ "recall≈0 是 LightGBM 校准效应"——半对，机制归因被我自己证伪

**判断（会话早期假说）**：头条 §2 的 recall≈0 是 LightGBM 校准到 5.4% 基率 → 阈值保守 → 移位后低 recall；换模型族不成立。

**对的半（确认）**：模型特异性成立——stump/LogReg 靠拉高 FPR 刷高 recall，LightGBM 给低 recall；recall@0.5 **跨模型不可比**。

**错的半（被 `frac∈[0.4,0.5)` 机制列证伪）**：若真是"基率校准→阈值保守"，攻击流概率该堆在 0.5 略下方；实测 `frac∈[0.4,0.5)≈0.000003` ——攻击流被**自信压到 0.4 以下**，不在阈值附近。两个反例钉死基率校准：①固定 1M 仅换 seed，recall 摆 8×（基率没变，校准位移该是常数）；②recall 随训练量非单调 0.019→0.21→0.29→0.0002（基率与 size 无关）。**真机制 = 操作点落在分数分布一个近空、随训练条件整体重组的稀疏尾区**，recall 测的是不稳定尾巴的瞬时大小。基率校准至多解释平均水平偏低，解释不了"为何这个数不可信"。

**复盘注意点（方法学母题）**：① 别停在"听起来合理"的机制故事上——**专门设计一个测量去证伪自己的假说**（这里特地加 `frac_40_50` 列）。结果它被证伪，逼出更强结论：recall 的失败**调阈值修不了**（比"校准保守"强，因为后者让人以为重设阈值能救）。② 推论：**操作点指标（recall@固定阈值）在分数稀疏区不可信**；不平衡安全数据优先**阈值无关 + rank-based**(PR-AUC)，且报告时连 PR-AUC 自己**贴基线处也会抖**（→UNSW CV 0.26–0.37）也要诚实给出。呼应 [[feedback-pr-auc-imbalanced]]。
